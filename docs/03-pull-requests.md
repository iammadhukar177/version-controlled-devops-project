# 🔁 Step 3: Pull Requests

Pull Requests (PRs) help ensure code is reviewed, tested, and safely merged.

---

## 🔄 Pull Request Workflow

1. Developer works on a `feature/` branch.
2. Push changes to GitHub.
3. Open a Pull Request (PR) to `dev` branch.
4. After testing and review, open another PR from `dev` to `main`.

---

## 🛠 GitHub PR Interface

- Assign reviewers
- Link to issues (if applicable)
- Add labels like `bug`, `enhancement`, etc.
- Use squash or rebase merge for clean history

---

## ✅ PR Tips

- Title should be clear and concise
- Add a meaningful description
- Link related issues or tasks
- Address review feedback
